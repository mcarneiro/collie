Collie - column module (IE9+)

This page contains example using collie for IE9+. It has one element less compared to collie legacy as it's possible to use calc().

Box modelling techniques

Default configuration using collie(). By default it has 2 columns, 10px gutter and uses float technique.

1
2
.example-default {
  collie();
}

The default mixin output will be:

.example-default {
  margin-left: -10px;
}

.example-default:after {
  clear: both;
  content: "";
  display: table;
}

.example-default > .col {
  box-sizing: border-box;
  margin-left: 10px;
  width: calc(50% - 10px);
}

The inline-block technique is good for multiple lines like product lists. By default it has vertical-align: top and a font-size: 0; on the main holder to avoid the space between elements.

The mixin output code is:

.example-ib {
  margin-left: -10px;
  font-size: 0;
  margin-top: -10px;
}
.example-ib > .col {
  box-sizing: border-box;
  margin-left: 10px;
  width: calc(50% - 10px);
  display: inline-block;
  font-size: 16px;
  margin-top: 10px;
  vertical-align: top;
}

An example using table technique, useful when an "auto" column is needed:

1 is
higher
2
.example-tb-auto {
  collie(4, $type: "table");
  margin-bottom: 10px;

  > .row-w > .col:first-child {
    width: auto;
  }
}

Notice that a .row-w element is needed as the table wrapper and .col will also be a wrapper to give the gutter space between each column.

The mixin output is:

.example-tb-auto {
  margin-left: -10px;
}

.example-tb-auto > .row-w {
  display: table;
  width: 100%;
}

.example-tb-auto > .row-w > .col {
  box-sizing: border-box;
  display: table-cell;
  padding-left: 10px;
  width: 25%;
}

.example-tb-auto > .row-w > .col:first-child {
  width: auto;
}

It's important to notice that the inheritance of the display table technique is tighter than others. To override display property in .col element, for example, you would need to use the selector .example-tb-auto > .row-w > .col { display: block; }

Playing with multiple column sizes

As this module just create the base box-modelling is up to you to define the column sizes of each element just by overriding the rules:

1 is
higher
2
3
.example-custom-sizes {
  collie();

  .module-a-wrapper { width: collie-width(percentage(1/5)); }
  .module-b-wrapper { width: collie-width(percentage(3.5/5)); }
  .module-c-wrapper { width: collie-width(percentage(0.5/5)); }
}

collie-width function will help on creating the calc value. The output for collie-width(percentage(1/5) would be calc(20% - 10px) as 10px is the default gutter.

An example with different gutter:

1 is
higher
2
3
.example-custom-sizes-gutter {
  $g = 20px;
  collie($gutter: $g);

  .module-a-wrapper { width: collie-width(percentage(1/5), $g); }
  .module-b-wrapper { width: collie-width(percentage(3.5/5), $g); }
  .module-c-wrapper { width: collie-width(percentage(0.5/5), $g); }
}

Responsive behaviour

The responsive behaviour, breakpoints and when to use them is up to you. You can use min-height to apply the modules and have all col elements stacked in mobile. If you are working with max-width you can call collie("reset"), that will make the elements blocked and full with a margin-top of $gutter.

Example with max-width:

1 is
higher
2
.example-max-width {
  collie();

  @media (max-width: $breakpoints.mobile-max) {
    collie("reset");
  }
}

Example with min-width:

1 is
higher
2
.example-min-width {
  @media (max-width: $breakpoints.mobile-max) {
    .col {
      margin-top: 10px;

      &:first-child {
        margin-top: 0;
      }
    }
  }

  @media (min-width: $breakpoints.tablet-min) {
    collie();
  }
}
